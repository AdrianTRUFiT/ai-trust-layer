# Canvas Case Note: Platform Dependency and AI Trust Infrastructure

## Purpose

This case note uses the recent Canvas LMS incident as a public example of a broader institutional trust problem.

This is not a claim that Canvas has no value.

This is not a replacement argument.

This is a dependency argument.

Canvas and similar platforms provide important learning, workflow, communication, and administrative value. The deeper issue is that many institutions now place too much operational trust inside a single vendor-controlled environment.

---

## The trust issue

When one platform holds identity, coursework, messages, records, workflow, and operational continuity, a disruption is not only technical.

It becomes institutional.

The question is not only:

> Was the platform breached?

The better question is:

> Why was so much trust concentrated in one place without an independent trust layer beside it?

---

## Why login security is not enough

A login can answer:

> Can this person enter?

It does not fully answer:

- Who acted?
- What changed?
- What was accessed?
- What was trusted?
- What was copied?
- What communication can be independently proven?
- What continuity exists if the vendor is unavailable?

Modern institutions need verification beyond platform access.

---

## AI-speed risk

AI-enabled fraud changes the risk profile after exposure.

If messages, names, emails, course context, institutional relationships, or internal timing become exposed, attackers may use that context to create more believable impersonation, phishing, and manipulation attempts.

The risk is not only stolen data.

The risk is trusted context being reused against the institution.

---

## The missing layer

Schools and organizations need a vendor-independent trust layer that can support:

- identity continuity
- communication proof
- access-event visibility
- platform dependency mapping
- emergency continuity snapshots
- independent audit trails
- recovery evidence
- human-readable trust reporting

This layer should sit beside platforms like Canvas, Google Workspace, Microsoft 365, PowerSchool, Blackboard, and other mission-critical systems.

---

## Keep the platform. Remove sole custody.

The strongest institutional path is not necessarily to abandon core platforms.

The stronger path is to stop allowing any one platform to become the only place where trust lives.

A platform can deliver workflow.

A trust layer preserves proof.

---

## Category statement

The Canvas incident is not only a cybersecurity story.

It is a trust infrastructure story.

One login is not enough.

Trust needs its own layer.
